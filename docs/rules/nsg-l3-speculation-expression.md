# nsg-l3-speculation-expression — 臆測の表現の回避

> ⚠️ ステータス: L3（LLM支援）・未実装。SDK が L3 を支持した後に実装予定。本ドキュメントは仕様の先行定義です。

- **ruleId**: `nsg-l3-speculation-expression`
- **nameJa**: 臆測の表現の回避
- **ガイドライン**: 日本語スタイルガイド 第3版 (`jtca-style-guide-3`)
- **レベル**: L3（LLM支援）
- **想定 severity**: warning
- **適用モード案**: 公用文 (`official`) / 学術 (`academic`)

## 書籍根拠

第3章 §3.7.7「臆測の表現を使用しない」（p. 101–102）

使用情報では「思われる」「〜かもしれない」「〜らしい」「たぶん〜」などの臆測の表現は使わず、はっきりと言い切る形にする。

## 動作原理

- **LLM への入力**: 文単位（文書の種類・モードの情報も付加）
- **判断基準**:
  1. 臆測・推量の表現（「〜と思われる」「〜かもしれない」「〜らしい」「おそらく〜」「たぶん〜」「〜のようだ」「〜と思う」）を含む文を検出する
  2. 文書の目的・モードを考慮する。公用文・技術文書（`official` / `academic`）では言い切り表現が求められるため指摘する
  3. 「事実として不確実であることを明示する必要がある文脈」（例：研究上の仮説提示、注意書きの中の可能性提示）では抑制する
- **出力**: 対象文の引用、臆測表現の箇所、言い切り表現への書き換え案

## なぜ L1/L2 で不可能か

「思われる」「〜かもしれない」などは正規表現で検出できるが（L1 でほぼカバー可能）、抑制すべきケースの判断が困難なため L3 に分類する。たとえば学術論文における「可能性がある」は研究上の謙虚さを示す正当な表現であり、公用文や取扱説明書での同じ表現とは区別して扱う必要がある。文書の性質と文脈を踏まえた判断が LLM により可能になる。

なお、よりシンプルな機械的パターンのみを対象とするなら L1 実装も検討できるが、抑制ロジックを含む完全な仕様は L3 とする。

## 例

**正例（指摘なし）**

> 異常があります。確認してください。

（言い切り表現。指摘なし）

**負例（指摘あり・公用文モード）**

> 異常があると思われます。

（臆測表現。「異常があります」への書き換えを提案）

**負例2（指摘あり・公用文モード）**

> この手順を省略すると、エラーが発生するかもしれません。

（「エラーが発生します」と言い切るよう提案）

## 偽陽性・運用注意

- 学術モード（`academic`）でも研究上の仮説・推測を明示する節（「〜と考えられる」「〜が示唆される」等）は慣用的に許容される。過度な指摘を避けるため severity を `info` 相当に下げることを検討する
- ブログ・SNS モードは適用しない（個人の意見表明・感想は臆測表現が自然）
- 「〜のように見える」「〜かと思います」などの丁寧表現との区別に注意
