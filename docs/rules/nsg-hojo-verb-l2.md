# nsg-hojo-verb-l2 — 補助動詞の仮名書き

- **ruleId**: `nsg-hojo-verb-l2`
- **ガイドライン**: 日本語スタイルガイド (`jtca-style-guide-3`)
- **レベル**: L2（形態素解析）
- **既定**: 有効 / severity = warning
- **適用モード**: 公用文 (`official`) / 学術 (`academic`) / ブログ (`blog`)

## 意図

日本語スタイルガイド 第3版 第1章「用字・用語」§1.2.2「品詞や意味による漢字とひらがなの使い分け」では、同じ動詞でも品詞・用法によって漢字とひらがなを書き分けることを求めている。

本動詞として用いる「見る」「置く」には漢字を、補助動詞として用いる「…てみる」「…ておく」にはひらがなを充てる例が示されている:

| 用法 | 表記例 | 品詞 |
| --- | --- | --- |
| 本動詞 | 図面を**見る**。 | 動詞 |
| 補助動詞 | 歩いて**みる**と、周辺の変化がよくわかった。 | 補助動詞 |
| 本動詞 | 右端に**置く**。 | 動詞 |
| 補助動詞 | 確保して**おく**。 | 補助動詞 |

また、付録1「漢字とひらがなの使い分け」では以下の補助動詞（補動）が仮名形を推奨形として列挙されており、「動詞として使う場合は漢字で書く」と注記されている。

本ルールは形態素解析によって接続助詞「て/で」の直後に来る漢字表記の動詞を検出し、仮名書きを提案する。

## 出典

日本語スタイルガイド 第3版（テクニカルコミュニケーター協会）第1章 1.2.2節「品詞や意味による漢字とひらがなの使い分け」

## 検出対象（補助動詞 辞書形 → 推奨仮名形）

| 漢字形（辞書形） | 推奨仮名形 |
| --- | --- |
| 上げる | あげる |
| 行く | いく |
| 致す | いたす |
| 頂く | いただく |
| 居る | いる |
| 置く | おく |
| 下さる | くださる |
| 来る | くる |
| 仕舞う / 了う | しまう |
| 成る | なる |
| 見せる | みせる |
| 見る | みる |
| 貰う | もらう |
| 遣る | やる |

## 例

| 区分 | テキスト | 期待 |
| --- | --- | --- |
| 正例 | `歩いてみると、周辺の変化がよくわかった。` | 指摘なし |
| 誤例 | `歩いて見ると、周辺の変化がよくわかった。` | 補助動詞「見る」を「みる」に変更する提案 |
| 正例 | `確保しておく。` | 指摘なし |
| 誤例 | `確保して置く。` | 補助動詞「置く」を「おく」に変更する提案 |

## nsg-koto-formalnoun との重複について

`nsg-koto-formalnoun` は名詞「事」（`pos="名詞"`, `pos_detail_1="非自立"`）を対象とする。  
本ルールは動詞（`pos="動詞"`）が接続助詞「て/で」に後続する場合のみを対象とし、名詞トークンには干渉しない。両ルールの対象品詞が異なるため重複はない。

## 実装メモ

- kuromoji による形態素解析で `pos="助詞"`, `pos_detail_1="接続助詞"` の「て/で」トークンを検出し、次トークンが `pos="動詞"` かつ補助動詞辞書（`basic_form` キー）にマッチする場合に提案を生成する。
- `basic_form`（辞書形）でマッチするため、活用形（「来た」→`基本形=来る`）にも対応。
- surface が全ひらがなの場合はスキップ（既に仮名書き済みの活用形を除外）。
- 本動詞用法（格助詞等の後に来る「見る」「置く」等）は直前に接続助詞が来ないため対象外。
- `lint()` は L2 ルールのため空配列を返す（morphological エンジン用）。
