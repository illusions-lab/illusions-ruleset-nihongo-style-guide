# nsg-l3-sentence-length — 1文の長さ（50字超え）

> ⚠️ ステータス: L3（LLM支援）・未実装。SDK が L3 を支持した後に実装予定。本ドキュメントは仕様の先行定義です。

- **ruleId**: `nsg-l3-sentence-length`
- **nameJa**: 1文の長さ（50字以内を推奨）
- **ガイドライン**: 日本語スタイルガイド 第3版 (`jtca-style-guide-3`)
- **レベル**: L3（LLM支援）
- **想定 severity**: warning
- **適用モード案**: 公用文 (`official`) / 学術 (`academic`) / ブログ (`blog`)

## 書籍根拠

第2章 §2.4.1「1文は50字以内を推奨する」（p. 70）

同節では、1文は50字以内にまとめることを推奨している。50字を超える場合は文を分割できないか検討し、名称・説明が並列して長くなる場合は箇条書きへの変換も有効とされる。

## 動作原理

- **LLM への入力**: 文書全体（または段落単位）
- **判断基準**:
  1. 文を句点（。）で区切り、各文の文字数を計測する
  2. 50字を超える文を候補として挙げる
  3. ただし、固有名詞・技術用語・引用を多く含む文など、分割の余地がない場合は指摘を抑制する
  4. 箇条書きへの変換が有効な場合はその旨を提案する
- **出力**: 対象文の引用、文字数、分割・箇条書き化の提案

## なぜ L1/L2 で不可能か

文字数の計測自体は L1 正規表現で可能だが、固有名詞の連鎖・技術用語の引用・意味的に不可分な節など、「分割できない理由」の判断には文脈理解が必要になる。機械的に50字超えを全件検出すると偽陽性が多発するため、L3 で文脈を踏まえた取捨選択を行う。

## 例

**正例（指摘なし）**

> インデントは、文末や文頭を任意の位置にそろえるための機能です。

（約35字。50字以内）

**負例（指摘あり）**

> 文末や文頭を任意の位置にそろえるというインデント機能を使って文章をブロック化すると、読みやすくなるように整えることができます。

（約65字。分割候補あり → 「インデントは〜の機能です。この機能で文章をブロック化し、読みやすく整えることができます。」）

## 偽陽性・運用注意

- コードブロック・ URL・固有名詞の長い列は文字数に加算してもよいが、分割指摘は行わない
- 箇条書き内の各項目はそれ自体が文でないため対象外
- 小説・SNS モードは適用しない（口語・文学的長文が許容される）
